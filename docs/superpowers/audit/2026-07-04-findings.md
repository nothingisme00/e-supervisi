# Lapisan: Keamanan (Audit Lapis 1)

## [Kritis] Kepala sekolah dapat membuka detail supervisi apa pun tanpa batas status/sekolah
- **Lokasi:** app/Http/Controllers/KepalaSekolah/EvaluasiController.php:41-54 (show); juga :56-73 (startReview), :75-126 (giveFeedback), :128-138 (complete), :140-164 (requestRevision)
- **Lapisan:** Keamanan
- **Dampak:** `show($id)` hanya memanggil `Supervisi::findOrFail($id)` tanpa filter status maupun `tingkat`. Dengan menebak/menaikkan `{id}`, kepala sekolah dapat membaca detail supervisi guru dari sekolah/tingkat lain dan supervisi berstatus draft (belum disubmit) — termasuk refleksi, proses pembelajaran, dan metadata dokumen. `downloadDocument` sengaja memaksa `user->tingkat === auth()->user()->tingkat`, membuktikan pemisahan tingkat memang diniatkan; `show()` melanggarnya (cross-user data leak). Selain itu `startReview/giveFeedback/complete/requestRevision` juga tanpa cek `tingkat`, dan `complete()` tanpa cek status sehingga bisa menyelesaikan supervisi milik sekolah lain di status apa pun.
- **Usulan perbaikan:** Tambahkan pembatasan `whereIn('status', [...submitted/under_review/revision/completed])` dan filter `tingkat` guru terhadap `auth()->user()->tingkat` pada semua method yang menerima `{id}` di EvaluasiController.
- **Status:** ditemukan

## [Tinggi] Dokumen supervisi tersimpan di disk publik dan ditautkan langsung, dapat diakses tanpa autentikasi
- **Lokasi:** app/Http/Controllers/Guru/SupervisiController.php:109,122 (disk 'public'); resources/views/guru/supervisi/detail.blade.php:148, view.blade.php:118, evaluasi.blade.php:123,232; resources/views/kepala/evaluasi/show.blade.php:159
- **Lapisan:** Keamanan
- **Dampak:** Dokumen diunggah ke `Storage::disk('public')` (storage/app/public, disimbolik ke public/storage) dan view menautkannya via `asset('storage/' . $dokumen->path_file)`. URL seperti `/storage/supervisi/{id}/{timestamp}_{jenis}.pdf` dapat diakses siapa saja tanpa login. Route download ber-auth (`admin.supervisi.download`, `kepala.evaluasi.download`) menjadi tak berarti karena preview memakai tautan publik langsung. Path sebagian dapat ditebak (id sekuensial, jenis_dokumen dari 7 tipe tetap, hanya timestamp yang acak) — dokumen ajar bisa bocor lintas pengguna.
- **Usulan perbaikan:** Simpan dokumen di disk privat (`storage/app`, bukan `public/`) dan sajikan hanya melalui route download/preview ber-auth yang sudah memverifikasi kepemilikan/otorisasi.
- **Status:** ditemukan

## [Tinggi] Dependensi runtime produksi memiliki advisori keamanan tingkat tinggi
- **Lokasi:** composer.lock — laravel/framework v12.36.1, symfony/http-foundation v7.3.5 (via `composer audit --format=plain`)
- **Lapisan:** Keamanan
- **Dampak:** `composer audit` menemukan 24 advisori pada 13 paket. Yang berdampak pada runtime produksi dan bersifat *high*: symfony/http-foundation CVE-2025-64500 (Incorrect parsing of PATH_INFO → limited authorization bypass; terpengaruh <7.3.7) dan laravel/framework CRLF injection di default email rule (terpengaruh <12.60.0; terpasang 12.36.1). Advisori *medium* runtime lain: Laravel Temporary Signed URL Path Confusion, guzzlehttp/psr7 (CRLF/host confusion), symfony/routing (URL injection), symfony/mailer & mime (header injection), league/commonmark, symfony/http-foundation SSRF; *low*: symfony/yaml (ReDoS/stack exhaustion), polyfill-intl-idn. (phpunit & psy/psysh bersifat dev-only.)
- **Usulan perbaikan:** Jalankan `composer update` untuk menaikkan laravel/framework ke >=12.61.1 dan symfony/* ke rilis patch (http-foundation >=7.3.7, dst.) lalu re-audit.
- **Status:** ditemukan

## [Sedang] Admin dapat menonaktifkan akunnya sendiri (self-lockout)
- **Lokasi:** app/Http/Controllers/Admin/UserController.php:224-233 (toggleStatus); app/Livewire/Admin/UserManagement.php:76-84 (toggleStatus)
- **Lapisan:** Keamanan
- **Dampak:** `destroy()` sudah mencegah admin menghapus dirinya sendiri (UserController.php:199), tetapi `toggleStatus()` (baik di controller maupun komponen Livewire) tidak memeriksa `$user->id === Auth::id()`. Admin bisa men-set `is_active = false` untuk akunnya sendiri; karena login menolak user tidak aktif (CustomLoginController.php:44), admin akan terkunci keluar. Terkait: `update()` juga tidak mencegah admin mengubah `role` dirinya sendiri menjadi guru/kepala sehingga kehilangan akses admin.
- **Usulan perbaikan:** Tolak `toggleStatus` (dan perubahan role) bila `$user->id === Auth::id()`.
- **Status:** ditemukan

## [Sedang] Cookie sesi tidak dipaksa `secure` + tidak ada header keamanan / lifetime tidak konsisten
- **Lokasi:** config/session.php:172 (`'secure' => env('SESSION_SECURE_COOKIE')`); .env.example:36-39; app/Http/Middleware/PreventBackHistory.php
- **Lapisan:** Keamanan
- **Dampak:** `SESSION_SECURE_COOKIE` tidak diset di .env.example dan default null → cookie sesi bisa terkirim lewat HTTP (bukan hanya HTTPS), rawan pencurian sesi bila TLS tidak dipaksa. Tidak ada header keamanan global (CSP, X-Frame-Options/clickjacking, HSTS); PreventBackHistory hanya menyetel header no-cache. Selain itu `.env.example` menyetel `SESSION_LIFETIME=120` sementara `config/session.php` default 10 dan SessionTimeout middleware timeout 10 menit — konfigurasi lifetime tidak konsisten/berpotensi membingungkan. (`http_only`=true dan `same_site`=lax sudah aman.)
- **Usulan perbaikan:** Set `SESSION_SECURE_COOKIE=true` untuk produksi, tambahkan header X-Frame-Options/CSP, dan selaraskan nilai lifetime sesi.
- **Status:** ditemukan

## [Rendah] `checkDocuments` tanpa cek kepemilikan (info disclosure minor)
- **Lokasi:** app/Http/Controllers/Guru/SupervisiController.php:182-191
- **Lapisan:** Keamanan
- **Dampak:** `checkDocuments($id)` hanya menghitung `DokumenEvaluasi::where('supervisi_id', $id)->count()` tanpa memfilter `user_id`. Guru mana pun dapat memanggil `/guru/supervisi/{id}/check-documents` untuk mengetahui jumlah dan status kelengkapan dokumen supervisi milik guru lain (kebocoran informasi terbatas; hanya angka).
- **Usulan perbaikan:** Tambahkan `Supervisi::where('id',$id)->where('user_id', auth()->id())->firstOrFail()` sebelum menghitung.
- **Status:** ditemukan

## [Rendah] Route sensitif tanpa rate limit
- **Lokasi:** routes/web.php:75 (upload), :80 (comment), :103/:128 (feedback)
- **Lapisan:** Keamanan
- **Dampak:** Hanya route login (throttle:5,1) dan change-password (throttle:10,1) yang di-rate-limit. Route upload dokumen, komentar, dan feedback tidak memiliki throttle sehingga terbuka untuk penyalahgunaan/spam/DoS ringan (misalnya unggah berulang atau banjir komentar). Catatan positif: route `logout` sudah benar berada DI LUAR grup throttle login.
- **Usulan perbaikan:** Terapkan `throttle` wajar (mis. `throttle:30,1`) pada route upload/comment/feedback.
- **Status:** ditemukan
